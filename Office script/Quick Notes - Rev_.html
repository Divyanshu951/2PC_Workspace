CSS Flexbox:

Sources: https://www.joshwcomeau.com/css/interactive-guide-to-flexbox/

When we flip display to flex, we create a “flex formatting context”. This means that, by default, all children will be positioned according to the Flexbox layout algorithm.
 

Flex direction:

  • It is for controlling the direction of the element in a row or a column, by default the items will stack side by side in a row but we can flip it to column using flex direction: column; property.
  • With flex-direction: row, the primary axis runs horizontally, from left to right.
  • When we flip to flex-direction: column, the primary axis runs vertically, from top to bottom.
The children will be positioned by default according to the following 2 rules:

Primary axis: Children will be bunched up at the start of the container.
Cross axis: Children will stretch out to fill the entire container.

Values:
        .container {
                flex-direction: row | row-reverse | column | column-reverse;
            }
  • row (default): left to right in ltr; right to left in rtl
  • row-reverse: right to left in ltr; left to right in rtl
  • column: same as row but top to bottom
  • column-reverse: same as row-reverse but bottom to top

Alignment

1. justify-content:

  • You can change how the items are distributed among the primary axis using the  justify-content property.
  • When it comes to the primary axis, we don't generally think in terms of aligning a single child. Instead, it's all about the distribution of the group.
  • We can bunch all the items up in a particular spot (with flex-start, center, and flex-end), or we can spread them apart (with space-between, space-around, and space-evenly).
Sure, here’s a quick revision of the flexbox alignment options:
        .container {
            justify-content: flex-start 
                             flex-end 
                             center 
                             space-between 
                             space-around 
                             space-evenly 
                             start 
                             end | left | right ... + safe | unsafe;
                    }
  • flex-start: Items start at the beginning of the flex-direction.
  • flex-end: Items align at the end of the flex-direction.
  • start: Items align at the start of the writing mode.
  • end: Items align at the end of the writing mode.
  • left: Items align to the left (if compatible with flex-direction).
  • right: Items align to the right (if compatible with flex-direction).
  • center: Items are centered along the line.
  • space-between: Items are spaced with the first at the start and the last at the end.
  • space-around: Equal space around items, with extra space on edges.
  • space-evenly: Equal space between items and container edges.

For the cross axis, things are a bit different. We use the align-items property:

2. Align-items: 

  • You can change how the items are distributed among the cross axis using the  justify-content property.
  • Same as justify-content but for cross axis.  Think of it as the justify-content version for the cross-axis.
Here's a brief revision for cross-axis alignment options:

  • stretch: Items stretch to fill the container (respects min/max-width).
  • flex-start / start / self-start: Items align at the start of the cross-axis (depends on flex-direction or writing-mode).
  • flex-end / end / self-end: Items align at the end of the cross-axis (depends on flex-direction or writing-mode).
  • center: Items are centered on the cross-axis.
  • baseline: Items align based on their text baselines.
It's interesting… With align-items, we have some of the same options as justify-content, but there isn't a perfect overlap.

3. Align-self:

  • Unlike justify-content and align-items, align-self is applied to the child element, not the container. It allows us to change the alignment of a specific child along the cross axis:
  • This allows the default alignment (or the one specified by align-items) to be overridden for individual flex items.
  • Only works on cross axis. 
        .container {
             align-self: auto | 
                         flex-start | 
                         flex-end | 
                         center | 
                         baseline | stretch;
                    }
Note that float, clear and vertical-align have no effect on a flex item.

align-self has all the same values as align-items. In fact, they change the exact same thing. align-items is syntactic sugar, a convenient shorthand that automatically sets the alignment on all the children at once.

let's give a proper definition to all 4 terms we've been talking about:
  • justify — to position something along the primary axis.
  • align — to position something along the cross axis.
  • content — a group of “stuff” that can be distributed.
  • items — single items that can be positioned individually.

  • Grid line:
In CSS Grid, grid lines are the dividing lines that create the structure of your grid layout. They are either:

  • Column grid lines (vertical lines that separate columns)
  • Row grid lines (horizontal lines that separate rows)
These lines are numbered, starting from 1 at the far left or top of the grid, and they help define where items will be placed.
